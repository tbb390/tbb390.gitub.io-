📝 刷题记录 · 第十天 【选题方案枚举】

题型：位运算 / 枚举 难度：中等 2026-05-16

📖 题目大意

本题给定 n 道题目 和 m 支参赛队伍。 每支队伍对每一道题是否通过是已知的（0 表示未通过，1 表示通过）。

现在需要从 n 道题中选择 10 ～ 13 道 组成一场比赛的题目集合， 并要求在该选题方案下：

  • 所有队伍按照通过题数从高到低排序
  • 指定排名位置的三支队伍，其通过题数必须 恰好等于给定值

如果存在满足条件的选题方案，则输出题目编号；否则输出 -1。

💡解题思路

  • 第一步：观察数据范围
    题目给出 n ≤ 13，这个规模非常关键。 如果 n 很大，就必须用贪心或动态规划； 但 n ≤ 13 意味着 —— 可以枚举所有题目组合。
  • 第二步：为什么用二进制枚举
    对于 n 道题，每一道题只有「选 / 不选」两种状态， 正好可以用一个二进制位表示：
    • 0 → 不选这道题
    • 1 → 选这道题
    因此一个长度为 n 的二进制数（mask）就代表一种完整选题方案。
  • 第三步：枚举所有可能方案
    所有方案一共有 2ⁿ 种， 当 n ≤ 13 时，最多只有 8192 种， 完全可以暴力枚举。
  • 第四步：剪枝（限制题目数量）
    题目要求必须选择 10～13 道题， 因此：
    • 先统计 mask 中 1 的个数
    • 不在区间内的方案直接跳过
    这一步可以大幅减少无效计算。
  • 第五步：统计每支队伍的通过题数
    对于当前选题方案：
    • 遍历所有队伍
    • 只统计被选中题目中，该队伍通过的数量
  • 第六步：排名判断
    将所有队伍的通过题数排序（从大到小）， 然后检查：
    • 第 rkg 名是否等于 pg
    • 第 rks 名是否等于 ps
    • 第 rkb 名是否等于 pb
    全部满足即为合法方案。

⚙️ AC 代码（C++）

#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(0);

    int n, m;
    cin >> n >> m;

    vector> a(m, vector(n)); // a[i][j] 表示第 i 支队伍是否通过第 j 题
    for (int i = 0; i < m; i++) {
        string s;
        cin >> s;
        for (int j = 0; j < n; j++) {
            a[i][j] = s[j] - '0';
        }
    }

    int rkg, rks, rkb;// 三个指定排名
    cin >> rkg >> rks >> rkb;

    int pg, ps, pb;// 对应排名要求的通过题数
    cin >> pg >> ps >> pb;

    for (int mask = 0; mask < (1 << n); mask++) {//枚举所有题目子集

        int cntProblem = 0;//统计当前方案选了多少题
        for (int j = 0; j < n; j++)
            if (mask & (1 << j)) cntProblem++;

        if (cntProblem < 10 || cntProblem > 13) continue; // 不满足题目数量要求，直接跳过

        vector solved(m, 0);// 统计每支队伍的通过题数
        for (int i = 0; i < m; i++)
            for (int j = 0; j < n; j++)
                if ((mask & (1 << j)) && a[i][j])
                    solved[i]++;

        sort(solved.begin(), solved.end(), greater());    // 按通过题数排序（降序）

        // 检查指定排名是否满足要求
        if (solved[rkg - 1] == pg &&
            solved[rks - 1] == ps &&
            solved[rkb - 1] == pb) {

            // 输出选中的题目数量
            cout << cntProblem << "\n";

            // 输出题目编号（注意空格控制）
            bool first = true;
            for (int j = 0; j < n; j++) {
                if (mask & (1 << j)) {
                    if (!first) cout << " ";
                    cout << j + 1;
                    first = false;
                }
            }
            return 0;
        }
    }

    // 所有方案都不满足
    cout << -1 << endl;
    return 0;
}
📎 易错点与细节说明
  • 排名是从 1 开始计数的，数组下标需要减 1
  • 一定要先判断题目数量是否在 10～13 之间
  • 输出题目编号时，最后一个数字后面不能有多余空格
  • 排序必须是降序，否则排名含义会出错

📚 今日总结

  • 当 n 很小（≤20）时，优先考虑二进制枚举子集
  • 位运算是表示「选 / 不选」最自然的方式
  • 枚举 + 剪枝是解决组合搜索题的经典套路
  • 排序后用下标直接判断排名
